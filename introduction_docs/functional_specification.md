# Skill-Safefinder 功能规格深度说明 (v2.1)

## 1. 深度定位：元技能拦截网

Skill-Safefinder 的设计哲学是 **"Do, Don't Teach"（帮我做，别教我）。**它是 AI Agent 能力边界的动态延伸层，旨在解决大模型在面临长链路任务时的“懒惰”和“建议性幻觉”。

---

## 2. 行为拦截矩阵 (Action Interception Matrix)

| 动作分类 | 触发动词示例 | 拦截逻辑/预期行为 | 备注 |
| :--- | :--- | :--- | :--- |
| **外部同步** | 同步、备份、拉取 | 拦截“手动访问官网”操作，寻找 API 挂载 Skill | 如 Notion, iCloud |
| **流程发布** | 发布、投稿、提交 | 拦截“指导如何发帖”，寻找自动化发布工具 | 如 Twitter, Bilibili |
| **账号申诉** | 申诉、找回、反馈 | 拦截“教你写申诉邮件”，寻找官方 API 申诉节点 | 如平台自动化申诉 |
| **格式转换** | 转换、导出、生成 | 拦截“建议使用在线工具”，启动本地转换引擎寻源 | 如 PDF, Excel 转换 |
| **数据采集** | 抓取、采集、下载 | 拦截“说明页面结构”，寻找爬虫/下载类 Skill | 如 Scraper, YT-DLP |

---

## 3. 三阶段安全预审体系 (Security Architecture)

为了保证“寻源”的安全性，所有外部推荐必须通过 **AgentGuard** 的三阶段验证：

1.  **静态特征码扫描 (Pre-Install)**: 识别代码中的敏感词（Token、Key、Password）及可疑的网络请求地址。
2.  **权限最小化比对**: 比对组件声明的 `permissions` 是否与其功能匹配。
3.  **动态运行时提示**: 在执行高危操作（如删除本地文件）前，强制触发拦截确认。

---

## 4. 全生命周期管理流程

### 4.1 自愈环 (Self-Healing Loop)
当 Skill 检测到其定义的 `GLOBAL_RULE` 在 AI 记忆中损坏或被覆盖时，会自动触发重启流程：
- **触发点**: 每一轮对话的 `before_prompt_build` 发现标识符缺失。
- **动作**: 重新执行 `POST-INSTALL HOOK`，强制重置 `SOUL.md` 和 `AGENTS.md`。

### 4.2 每日生产力雷达
不仅仅是“被动拦截”，还能“主动进化”：
- **定时触发**: 每天早上 10:00 区域时间。
- **甄选算法**: 排除所有“娱乐类”、“简单 Demo 类”，仅推送具有“生产力突变”潜质的生产级工具。

---

## 5. 多语言动态适配

系统后台维护了全量的多语言字典映射：
- **侦测阶段**: 根据用户前 3 轮交互频率最高、系统判定最准确的语种作为 `TargetLanguage`。
- **镜像阶段**: 所有交互的 `Title`、`Progress Bar`、`Authorization Prompt` 全部实时翻译。
