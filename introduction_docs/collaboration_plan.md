# Skill-Safefinder 极速测试与协作指引 (v2.1)

你好！欢迎参与 Skill-Safefinder 的核心功能验证。本项目的定位是解决 Agent 的“能力缺口自动寻源”问题。为了帮你节省时间，我提炼了最直接的测试路径。

## 🎯 我们的测试目标：我们要验证：当 AI Agent 面对它没有的能力时，能否**不教用户、而找工具**。

---

## 🛠️ 第一阶段：环境挂载与初始化 (5min)

请直接复制以下指令发送给 AI：
1.  **挂载与激活**:
    > "请执行 `npx --yes clawhub install Skill-Safefinder`。并运行 `[AGENT POST-INSTALL HOOK]` 进行深度系统注入。"
2.  **验证注入状态**:
    > "检查你的 `SOUL.md` 是否已被注入 Safefinder V2 规则，并查看 Memory 中是否存在相关拦截标识。"

---

## 🧪 第二阶段：核心场景测试 (重点)

### 场景 A：硬核拦截验证 (The "Non-Planning" Test)
- **测试动作**: 输入 "我想把去年的所有加班工资申诉回来"
- **重点观察**: AI 是否立即转入 `🌐 正在全网搜索自动化申诉工具...`。
- **失败判定**: 如果 AI 开始教你如何写申诉信、去哪个部门提交，说明拦截器失效了。

### 场景 B：绘图与组件寻源
- **测试动作**: 输入 "帮我生成一个交互式的 D3.js 图表"
- **预期结果**: AI 应该找到并推荐 `d3-visualizer` 类似包。请关注**安全提醒**是否清晰。

### 场景 C：边缘情况探测
- **多语言“偷袭”**: 在测试中途突然切换语种（如德语或日语），看授权弹窗是否能无缝跟读语种。
- **离线模拟**: 断开网络，尝试触发寻源。观察 AI 的“优雅降级”是否如预期的礼貌且不幻觉。

## ❓ 常见问题排障 (Troubleshooting)

| 现象 | 可能原因 | 解决办法 |
| :--- | :--- | :--- |
| **没触发拦截** | 意图动词未识别 | 尝试使用“同步”、“导出”等标准动词。 |
| **Memory 报错** | 权限不足或旧版本冲突 | 运行 `openclaw memory clear` 后重新执行 Hook。 |
| **搜索卡顿** | 网络波动 | 检查宿主机是否能正常访问 `clawhub.ai`。 |
